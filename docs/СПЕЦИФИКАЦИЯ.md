# Спецификация проекта «Очистка хранилища macOS»

Версия документа: **0.2**  
Статус: **черновик**

## 1. Цель продукта

Дать пользователю macOS удобный способ **понять**, что занимает место на диске, и **безопасно** освободить пространство (в первую очередь через осознанные действия пользователя, а не «молчаливую» очистку всего подряд).

## 2. Не-цели (на текущий этап)

- Автоматическое удаление системных файлов без явного подтверждения.
- Гарантия освобождения конкретного объёма места.
- Поддержка платформ, отличных от macOS.

## 3. Архитектура репозитория

| Модуль       | Ответственность |
|-------------|-----------------|
| `CleanerCore` | Доменные типы, сценарии (use cases), политики; **без** SwiftUI. |
| `CleanerUI`   | Представления SwiftUI, завязанные на публичные модели/API из Core. Исходники внутри: `AppShell/` (каркас окна), `Features/<Имя>/` (экраны и заглушки). |
| `CleanerApp`  | Точка входа приложения, композиция сцен и окон. |

Правило зависимостей: **Core не зависит от UI**.

### 3.1 Раскладка `Sources/CleanerUI/`

| Каталог | Назначение |
|---------|------------|
| `AppShell/` | Оболочка главного окна, навигация (`NavigationSplitView` и т.п.), общие элементы боковой панели. |
| `Features/<Раздел>/` | Конкретный раздел: экраны, заглушки, в будущем — ViewModel только для этого раздела (по мере роста). |
| `RootView.swift` | Публичная точка входа UI для `CleanerApp`. |

## 4. Фазы (высокий уровень)

| Фаза | Содержание |

| 0    | Инициализация репозитория, модулей, спеки, правил, документации. |

## 5. Критерии приёмки для фазы 0

- [x] Репозиторий содержит `Package.swift`, модули `CleanerCore`, `CleanerUI`, исполняемый таргет `CleanerApp`.
- [x] Команда `swift build` завершается успешно на заявленной версии macOS/Xcode (см. README).
- [x] В корне есть `README.md`, `LICENSE` (MIT), `CONTRIBUTING.md`, `SECURITY.md`, `CHANGELOG.md`.
- [x] Описан подход «спека → код» и расположение спецификации (`docs/СПЕЦИФИКАЦИЯ.md`).
- [x] Настроены рекомендации для агента Cursor (`.cursor/rules/`).

## 6. Критерии приёмки для фазы 1 (каркас UI)

- [x] Одно главное окно (`WindowGroup` + корневой `RootView`).
- [x] `NavigationSplitView` с боковой панелью и не менее чем **двумя** разделами (реализовано три: «Сканирование», «Настройки», «О программе»).
- [x] У каждого раздела — заглушка: заголовок и короткий текст «скоро».
- [x] Исходники в `CleanerUI` разложены по `AppShell/` и `Features/<раздел>/` согласно п. 3.1.
- [x] Нет логики сканирования ФС, удаления и запросов Full Disk Access.
