# Спецификация проекта «Очистка хранилища macOS»

Версия документа: **0.9**  
Статус: **черновик**

## 1. Цель продукта

Дать пользователю macOS удобный способ **понять**, что занимает место на диске, и **безопасно** освободить пространство (в первую очередь через осознанные действия пользователя, а не «молчаливую» очистку всего подряд).

## 2. Не-цели (на текущий этап)

- Автоматическое удаление системных файлов без явного подтверждения.
- Гарантия освобождения конкретного объёма места.
- Поддержка платформ, отличных от macOS.

## 3. Архитектура репозитория

| Модуль       | Ответственность |
|-------------|-----------------|
| `CleanerCore` | Доменные типы, сценарии (use cases), политики; **без** SwiftUI. Поверхностный листинг: `DirectoryListing/` (в т.ч. `FolderPathScope`), оценка занятого места по категориям: `StorageScan/`, классификация путей и быстрая очистка: `Safety/`, агрегация для обзора диска: `DiskOverview/`, форматирование размера: `Formatting/`, сведения о продукте: `AppMetadata`. |
| `CleanerUI`   | Представления SwiftUI, завязанные на публичные модели/API из Core. Исходники внутри: `AppShell/` (каркас окна), `Features/<Имя>/` (экраны и заглушки). |
| `CleanerApp`  | Точка входа приложения, композиция сцен и окон. |

Правило зависимостей: **Core не зависит от UI**.

### 3.1 Раскладка `Sources/CleanerUI/`

| Каталог | Назначение |
|---------|------------|
| `AppShell/` | Оболочка главного окна, навигация (`NavigationSplitView` и т.п.), общие элементы боковой панели. |
| `Features/<Раздел>/` | Конкретный раздел: экраны, заглушки, в будущем — ViewModel только для этого раздела (по мере роста). |
| `RootView.swift` | Публичная точка входа UI для `CleanerApp`. |

### 3.2 Дистрибуция (пользовательские сборки)

- Для установки без Xcode публикуется **GitHub Release** с ZIP-архивом, внутри которого **`MacosStorageCleaner.app`** (минимальный bundle: исполняемый файл + `Info.plist`, собранные скриптом `scripts/package-macos-app.sh`). Для прямой ссылки «скачать последнее» к релизу прикладывается **`MacosStorageCleaner-macos.zip`** (одинаковое имя в каждом релизе); дублируется архив с версией в имени (`MacosStorageCleaner-<версия>-macos.zip`).
- Маркетинговая версия в интерфейсе и в имени архива совпадает с **`AppMetadata.marketingVersion`**; CI при push git-тега `v*` прикрепляет артефакт к релизу (см. `.github/workflows/release.yml`).
- Код подписи Apple для открытого репозитория не предполагается; пользователь подтверждает первый запуск средствами macOS (см. README).

## 8.1 Два режима сканирования (фаза 4 / #6)

| Режим | Корень обхода | Назначение |
|--------|----------------|------------|
| **Папка** | Каталог, выбранный через `NSOpenPanel` | Точечный анализ выбранного поддерева. |
| **Весь диск** | Корень **`/`** (смонтированный системный том) | Обзор верхнего уровня диска и навигация вглубь по тем же правилам, что и в режиме папки, без обязательного предварительного выбора папки. |

Оба режима используют один и тот же поверхностный листинг и один UI-раздел «Сканирование»; режим переключается явной командой пользователя.

## 8.2 Права macOS, песочница и честные ограничения

- Приложение **не обещает** полный обзор системы без разрешений ОС: часть каталогов может быть недоступна для чтения.
- Для чтения за пределами типичных пользовательских областей пользователю может понадобиться **«Полный доступ к диску»** (Системные настройки → Конфиденциальность и безопасность). В UI показывается **понятное объяснение**, если перечисление каталога завершилось ошибкой доступа.
- В спецификации зафиксировано: **нет автоматического запроса TCC** из кода в MVP; пользователь добавляет приложение вручную при необходимости.
- Сборка через SwiftPM **не добавляет** отдельный профиль песочницы entitlements в репозитории; фактические ограничения доступа зависят от способа подписи/упаковки продукта. Поведение приложения при отказе в доступе должно оставаться предсказуемым (сообщение об ошибке, без падения).

## 8.3 Быстрая очистка (#6)

- Отдельный раздел UI **«Быстрая очистка»**: операция запускается **только** после явного действия пользователя.
- Перед применением — **диалог** со списком затрагиваемых **классов** путей и **оценкой** объёма там, где она возможна **без глубокого рекурсивного скана** (поверхностный подсчёт по первому уровню вложенности внутри известных корней категорий).
- После построения плана пользователь может **снять отметки** с отдельных верхнеуровневых объектов (галочки): исключение из плана **не удаляет** ничего на диске; в «Выполнить» попадают только оставшиеся отмеченные пути.
- Категории MVP (только согласованные и относительно безопасные корни):
  1. **Кэши пользователя** — верхний уровень `~/Library/Caches` (каждый непосредственный элемент переносится в **корзину** отдельно).
  2. **Временные файлы пользователя** — содержимое каталога временных файлов процесса (`NSTemporaryDirectory()` и эквивалент для текущего пользователя), перенос в корзину по элементам первого уровня.
  3. **Корзина** — **безвозвратная** очистка содержимого пользовательской корзины (`~/.Trash`) как исключение: элементы уже в корзине; операция выполняется **только** после **отдельного усиленного подтверждения** с явным текстом о безвозвратности (не «перенос в корзину»).
- **Запрещено** для быстрой очистки: любые пути вне разрешённых корней категорий; системные тома (`/System`, системные `/Library`, `/private/var/db`, цепочки ключей и т.д.) — **блокируются** в **CleanerCore** с понятной причиной.
- **Критерий приёмки:** отмена на любом шаге диалога **не меняет** файловую систему; по завершении — сводка «выполнено / пропущено» с причиной для пропусков.
- **Не-цель:** обещание конкретного освобождённого объёма; «тихое» удаление без согласия пользователя.

## 8.4 Эвристики «кандидат на очистку» и «важно»

- Правила классификации путей живут в **CleanerCore** (`PathSafetyClassifier`); UI **только отображает** метку/стиль.
- Подсветка «относительно безопасный кандидат» — **рекомендация, не гарантия**; в интерфейсе всегда показывается дисклеймер.
- «Важно»: системные области, связки ключей, внутренности `.app`, пользовательские медиа и документы по эвристике (не исчерпывающий список, уточняется в коде Core).
- Для путей из **запретного** набора операции быстрой очистки и переноса в корзину из этого MVP **заблокированы** с объяснением.
- Опционально в «Настройках»: **отключить** подсветку эвристик (только отображение, логика классификации в Core остаётся доступной для будущих сценариев).

## 8.5 Визуализация (лёгкая, без лишнего шума)

- Небольшой блок **диаграммы** (столбчатая или иная простая форма) по **уже полученному** списку текущего уровня: если текущий каталог **`/`**, элементы агрегируются в верхнеуровневые **категории обзора**: «Пользователь» (`/Users`), «Приложения» (`/Applications`), «Система» (`/System`, `/usr`, `/bin`, `/sbin`, `/etc` и т.п.), «Данные» (`/Library`, `/private`, `/var`, `/Volumes`, прочие служебные вершины), «Прочее`.
- Если данных для диаграммы недостаточно (например, не корень `/` или пустой список) — показывается ненавязчивая заглушка («нет данных для диаграммы» / «ещё считается» при загрузке).
- **Индикатор хода** поверхностной загрузки: компактная **sparkline** или эквивалент на основе хода async-операции листинга **без** отдельного тяжёлого прохода по диску.

## 8.6 Автоанализ памяти и UX-тексты (итерация после #6)

- При открытии раздела **«Память»** (если включено в настройках) выполняется **фоновый анализ** типичных категорий (программы, загрузки, документы, рабочий стол, фото, видео, музыка, кэши, среда разработки, корзина) с **индикатором прогресса** и круговой диаграммой по результатам.
- Если автоанализ **выключен**, при открытии показывается **последний сохранённый снимок** оценки по категориям; новый подсчёт — по кнопке **«Обновить оценку»**.
- На экране **«Память»** отображаются **занятое, свободное и общее** место на томе домашней папки (по данным macOS, в т.ч. «важное» свободное место, если система его отдаёт).
- После **быстрой очистки** в UserDefaults сохраняется **оценка** освобождённого объёма (по известным размерам целей плана) и **накопительная** оценка; экран «Память» обновляет блок истории при следующем появлении и по внутреннему уведомлению после очистки.
- Пользовательские тексты в UI — **бытовые формулировки** без обязательного показа системных путей; служебные пути — **опционально** (настройка).
- **Быстрая очистка** дополняется поиском **крупных файлов и папок** (подсказка, не полный обход диска) и действием «Показать в Finder».
